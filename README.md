# ES6

## var let const

- 重新声明

`var` : 在相同的作用域内可以多次声明同一个变量,后面的声明会覆盖前面的声明
`let和const` : 在相同的作用域或块级作用域内不允许重新声明已经存在的变量

- 重新赋值

`var和let`: 使用这两个关键字声明的变量可以在后续被重新赋值
`const` : 声明的变量必须在声明时初始化,并且一旦赋值后不能被重新赋值.

- 提升

`var` : 无论在函数的哪个位置声明变量,变量都会被视为在函数的顶部声明,如果变量没有初始化,则被视为undefined
`let和const` : 虽然技术上也会提升,但是被限制在一个称为暂时性死区的区域,直到实际的代码被执行并且变量被声明

- 作用域

`var` : 声明的变量具有函数作用域,在整个包含函数内部都可见,如果在函数外部使用var声明变量,该变量具有全局作用域
`let和const` : 这两者声明的变量都有块级作用域,只在包含他们的代码块内部可见
